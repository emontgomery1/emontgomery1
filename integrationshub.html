Step-by-Step Guide to Integration Setup

Last updated: November 14th, 2019

Integrations Overview

The Integrations Hub gives major-life purchase marketers the ability to automate the connection between Jornaya’s Activate data and marketing technology platforms. It is now easy to quickly activate and optimize campaigns using this data, without the need for technical resource investment.

Requirements

There are 3 requirements for setting up an integration:

  1. Platform Authentication - the Integrations Hub needs access to your platform and your Jornaya SFTP account in order to get and/or post data. We would suggest leveraging a generic user account (i.e. integrationuser@companyname.com) vs. a personal user account so multiple users within your organization can access and maintain the integrations.
  2. Common Identifier - The identifier used in the “recordid” column of the Activate monitoring file needs to be available in the platform being integrated with Activate.
  3. Data Mappings - Fields within your platform will need to be mapped to Jornaya schema, such as our monitoring file schema for posting data into Jornaya and the journeys we are posting into your platform.

Integration Types

Regardless of the platform, there are 2 primary ways to integrate with Jornaya Activate

  1. Deliver Monitoring Data to Jornaya
  2. Deliver Jornaya Signal to Your Platform

Deliver Monitoring Data to Jornaya

This type of integration enables major-life purchase marketers to easily post their portfolio data into Jornaya Activate to monitor for in-market activity. Within the Integrations Hub, users can map the identifiers from their marketing platform to the required Jornaya monitoring fields.

The Integrations Hub also has the ability to hash identifiers being sent to Jornaya within the monitoring file, specifically phone numbers and email addresses. These identifiers will be hashed using the SHA256 algorithm.

This type of integration will only run once. To post updated monitoring data to Jornaya, simply delete the current setup and create a new setup.

Deliver Jornaya Signal to Your Platform

This type of integration automatically ingests Jornaya in-market signal data into any platform that can use the data for marketing outreach. Within the Integrations Hub, you will need to map each Journey (i.e. Mortgage - Purchase, Insurance - Auto, etc.) to the field in your platform where that in-market signal value (i.e “Low to High”) should live.

These integrations are setup to run daily.

Getting Started

Prerequisites

All of our Integrations either pull monitoring data from fields in your platform or post Activate signal to fields in your platform. Before moving forward with setting up an integration, please make sure you know which fields you plan to pull data from and/or have created fields where the Activate data should end up within your platform(s).

Accessing the Integrations Hub

If you are interested in using the Integrations Hub, let your Jornaya Client Success Manager (CSM) know which integration you’d like to use. If the integration is available, you will receive an email to create your Integrations Hub account.

1. Click on the link in the email and login using the provided username & temporary password.You will be prompted to change the password; save this password as you will need this to access the integrations hub in the future

  • Minimum 8 characters
  • At least one capital letter
  • At least one lower case letter
  • At least one numeric value
  • At least one special character

2. Login with the updated password.

3. Select the integration you'd like to begin setting up.

Jornaya Signal to Velocify

Velocify Integration Overview

The integration posts in-market signal to mapped fields on the user form in Velocify. The integration can also:

  • Post last updated timestamps to mapped fields on the user form in Velocify
  • Map in-market signal fields & value combinations to actions created in Velocify (i.e. Insurance-General: High triggers an in-market insurance action)

Velocify Integration Details:

  • Users will authenticate with their Jornaya SFTP account & Velocify Account.
  • To use this integration, the recordid in your Activate monitoring file must contain only Velocify lead ids.
  • Once the integration is setup, users must click the yellow play button, which will enable the integration and display of a green checkmark.
  • The first run of the integration will start 5 minutes after the integration is enabled and will post the file from that same day.
  • After the first run, the integration will post files each morning when the signal file is available.

Authentication

  1. Navigate to the Integrations Hub homepage and login with your credentials
  2. Click the Blue Arrow next to the “Deliver Signal to Velocify” option
  3. Authenticate with your Jornaya SFTP Account. (This will be a popup window.)
    • Enter any name you'd like for the Authentication name and select SFTP in the dropdown.
    • Enter sftp.jornaya.com for the host and the username and password provided via email by Jornaya Support. (Username and password are case sensitive.)
    • Leave Private Key and Passphrase blank.
    • Port number should always be 22 and cipher should be left as default.
  4. Authenticate with your Velocify account.

Field Mapping

  1. Enter your Jornaya SFTP username again
  2. In the Jornaya Field to Velocify Field Mapping screen, map the Jornaya journeys you’ve purchased to fields on the Velocify customer record. (These fields have type ahead capability).
  3. In the Velocify field to Velocify action mapping screen, you can optionally setup a mapping for a journey/signal combination to an action with the same name. (The fields have type ahead, except the value field). For example:
    • Velocify field = Mortgage - Purchase
    • Value = High
    • Velocify Action: In-Market Signal Mortgage - Purchase = High
  4. Select Finish.

Enable

The integration is now setup, but not enabled. If changes need to be made, you will need to click the red trash can icon and walk through the setup again.

  1. By clicking the yellow play button, the integration will be enabled and turn to a green checkmark. The integration will run 5 minutes after enabling the integration, using today’s Activate dataset.
  2. To pause the integration, log back into the integration hub, hover over the green checkbox and click the yellow pause icon.
  3. To change the integration setup, simply delete the integration instance by clicking the red trash can icon and starting over.
